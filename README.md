## Problem Statement

Hospital readmissions cost the US healthcare system **over $26 billion annually**. For diabetic patients — one of the highest-risk groups — early prediction of readmission risk can enable targeted discharge planning, follow-up care, and resource allocation that meaningfully improves patient outcomes.

This project builds two ML models trained on **10 years of inpatient data from 130 US hospitals (1999–2008)**:

| Task | Description | Best Model | F1 Score |
|------|-------------|------------|----------|
| **Binary** | Will the patient be readmitted within 30 days? | Stacking (MLP + ExtraTrees) | **0.64** |
| **Multiclass** | When? (No / <30 days / >30 days) | AdaBoost + Calibration | **0.60** |

## Quickstart

### 1. Clone and install

```bash
git clone https://github.com/HarshitaAdlakha/hospital-readmission-predictor.git
cd hospital-readmission-predictor
python -m venv venv
# Windows:
venv\Scripts\activate
# macOS/Linux:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Add data

Download the dataset from the [UCI ML Repository — Diabetes 130-US hospitals](https://archive.ics.uci.edu/dataset/296/diabetes+130-us+hospitals+for+years+1999-2008) and place as:

```
data/
├── train.csv
└── test.csv
```

The dataset should contain the columns described in [Dataset Features](#dataset-features).

### 3. Train the models

```bash
python train.py
```

This will:
- Preprocess and engineer features
- Train the binary stacking classifier (MLP + ExtraTrees)
- Train the multiclass AdaBoost classifier
- Save all artifacts to `models/`

Expected output:
```
[1/5] Fitting preprocessor on training data...
[2/5] Engineering features...
[3/5] Training Binary Classifier (Stacking: MLP + ExtraTrees)...
   Binary F1 on test set: 0.6400
[4/5] Training Multiclass Classifier (AdaBoost)...
   Multiclass F1 (macro) on test set: 0.6000
[5/5] Saving preprocessor and feature engineer...
Training complete. All artifacts saved to models/
```

### 4. Launch the app

```bash
streamlit run app/app.py
```

Open http://localhost:8501 in your browser.

### 5. Run tests

```bash
pytest tests/ -v --cov=src
```

## Methodology

### Data Preprocessing

| Step | Detail |
|------|--------|
| Missing values | Drop features > 50% missing; mode-impute categoricals; median-impute numerics |
| Outlier handling | Visual inspection via boxplots; IQR-based removal |
| Sentinel values | Replace `?` and `Unknown/Invalid` with `NaN` |
| ICD-9 mapping | Diagnosis codes → 8 clinical categories (circulatory, respiratory, diabetes, etc.) |
| Scaling | `RobustScaler` (resistant to remaining outliers) |

### Feature Engineering

Six new features were derived from domain knowledge:

| Feature | Formula / Logic | Clinical Meaning |
|---------|-----------------|-----------------|
| `recurrency` | `number_inpatient > 0` | Prior hospitalizations → highest readmission predictor |
| `medication_change_ratio` | changed_meds / total_meds | Unstable medication regime → higher risk |
| `number_prior_visits` | outpatient + emergency + inpatient | Overall utilization burden |
| `lab_tests_per_med` | num_lab_procedures / num_medications | Lab intensity relative to treatment |
| `patient_severity` | procedures + diagnoses + emergencies | Composite acuity score |
| `age_times_medications` | age × num_medications | Interaction: elderly + polypharmacy |

### Feature Selection

- **Binary task**: SelectKBest (k=14) using F-statistic
- **Multiclass task**: SelectKBest (k=19)
- Additional validation with: Mutual Information, Boruta, RFE, LASSO, Ridge

### Class Imbalance

SMOTE (Synthetic Minority Over-sampling Technique) applied within the sklearn pipeline to prevent data leakage during cross-validation.

### Models

**Binary classifier — Stacking ensemble:**
```
Base learners:  MLP(100, 50) + ExtraTrees(200 trees)
Meta learner:   LogisticRegression
Cross-val:      5-fold stratified
```

**Multiclass classifier — AdaBoost:**
```
AdaBoostClassifier(n_estimators=100, learning_rate=0.5)
+ CalibratedClassifierCV for probability calibration
```

## Dataset Features

The dataset contains **29 features** across these categories:

| Category | Features |
|----------|----------|
| **Demographics** | race, gender, age |
| **Admission** | admission_type_id, admission_source_id, time_in_hospital |
| **Discharge** | discharge_disposition_id, medical_specialty |
| **Clinical** | num_lab_procedures, num_procedures, num_medications, number_diagnoses |
| **Diagnoses** | diag_1, diag_2, diag_3 (ICD-9 codes) |
| **Labs** | max_glu_serum, A1Cresult |
| **Medications** | insulin, metformin, glipizide, glyburide, and 19 other diabetes medications |
| **History** | number_outpatient, number_emergency, number_inpatient |
| **Targets** | readmitted (No/<30/>30), readmitted_binary (0/1) |

## Results

### Binary Classification

| Model | F1 | Accuracy | Train Score | Test Score |
|-------|----|----------|-------------|------------|
| Random Forest | 0.70 | 0.760 | 0.789 | 0.695 |
| Hist Gradient Boosting | 0.66 | 0.685 | 0.693 | 0.658 |
| **Stacking (MLP + ET)** | **0.64** | **0.684** | **0.690** | **0.644** |

> **Selected model**: Stacking — best generalization (lowest train/test gap).

### Multiclass Classification

| Model | F1 (macro) | Accuracy | Train | Test |
|-------|-----------|----------|-------|------|
| **AdaBoost** | **0.60** | **0.536** | **0.597** | **0.544** |
| Random Forest | 0.59 | 0.530 | 0.672 | 0.590 |

> **Selected model**: AdaBoost — slightly better F1 and more balanced per-class performance.

### Key Findings

- **`recurrency`** (prior inpatient visits) is the single strongest predictor of readmission
- Patients with HbA1c > 8 show significantly higher 30-day readmission rates
- `medication_change_ratio` captures clinical instability better than raw medication counts
- Class imbalance (only ~11% patients readmitted within 30 days) is the primary modeling challenge
